# Create Pull Request

## Overview

Creates or updates a PR for the current branch with an auto-generated description following conventional commit format.

## Steps

Run these **in parallel** to gather context:
- `git branch --show-current`
- `git status --short`
- `git diff <base-branch>...HEAD`
- `git log <base-branch>..HEAD --oneline`
- `gh pr list --head <current-branch>` (check if PR already exists)

### 1. Handle uncommitted changes on master/main

If currently on master/main with uncommitted changes:
- Analyze changes to auto-generate a branch name (e.g., `feat/add-user-auth`, `fix/login-validation`)
- `git add -A`
- `git checkout -b <feature-branch>`
- Commit with a one-liner conventional commit message (no footer)
- `git branch -f master origin/master` (or main)

### 2. Push if needed

If the branch has no remote or is ahead: `git push -u origin <current-branch>`

### 3. Process arguments

Arguments: `$ARGUMENTS`
- Quoted string = rationale (proofread it: fix typos, preserve intent)
- Unquoted word = base branch (default: `master`)

### 4. Generate PR title and description

Analyze all commits and changes:
- Title: conventional commit format, explains "why" not just "what"
- Summary: 2-4 bullets describing the changes
- Rationale section: only if provided

### 5. Look for resolved issues

Run `gh issue list --state open`. Add `Fixes #<number>` to body for each issue resolved by the PR.

### 6. Create or update PR

Use HEREDOC to pass body:

```bash
# New PR
gh pr create --draft --title "<title>" --body "$(cat <<'EOF'
## Rationale
<proofread rationale, only if provided>

## Summary
- First change
- Second change

Fixes #123
EOF
)"

# Existing PR
gh pr edit --title "<title>" --body "$(cat <<'EOF'
...
EOF
)"
```

Return the PR URL.

## Rules

- Default base branch: `master`
- Always open new PRs as **draft**
- No "Generated by" footers
- No "Co-Authored-By" in commit messages
